# Participle Layer — meaning beside the clause

Participles are **read-only readings** next to the finite clause (or verbless
verse) they belong with. Morphology finds them (Brick 4). Structure shows what
the form means and how it hangs on the clause — not a bare inventory that they
exist. There is no student sorting workflow and no aggregate “Participle Views”
page.

---

## 1. Finding participles (mechanical)

Morphology already tags mood. Brick 4 in the Greek O-Prototype marks participle
tokens — pure lookup, no judgment. Structure trusts those marks.

## 2. Relating a participle to a clause

Same-verse attachment only:

1. If the participle’s Spanish word sits inside a saved finite-clause span in
   **the same verse**, attach it to that clause.
2. Else — do **not** nearest-neighbor attach to another span in the verse
   (that falsely hung 1 Pet 1:7’s «oro» participles on «sea hallada»). Show
   under Skeleton as verse orphans (or under verbless verses when there is no
   finite verb). Never steal a participle into a later verse’s finite clause
   (e.g. 1 Pet 1:3 must not attach to 1:6).

No attributive / substantival / circumstantial sort. No semantic-relation quiz.

## 3. Visual design

**A. Underline-in-place.** Confirmed participles keep a distinct dotted underline
inside the clause’s displayed Spanish text.

**B. Meaning beside the host (active clause card).** Spanish · Greek, form line
(aspect · voice · case), hang line from case / agreement.

**C. Skeleton / clause card: host first.** When morphology finds an agreeing
noun or pronoun, group under that host:

```
oro
  → perece · ἀπολλυμένου
  → probado · δοκιμαζομένου
```

Nominative participles with no auto host ask **Who do they ride with?** — tap
to pick the subject in the Spanish context. The picker includes the **whole
chapter** (so 1:8 visto can hang under 1:7 Jesucristo). After picking:

```
Jesucristo
  → visto · ἰδόντες
```

## Non-goals

- Participles never become finite-clause tree nodes or change indent depth.
- No Participle Views (Flow / Emphasis / Cast).
- No Continue / unsorted gauntlet and no localStorage “observations” for sort.
- No automatic *semantic* classification (when / why / how).
